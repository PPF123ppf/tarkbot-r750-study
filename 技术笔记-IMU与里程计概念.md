# 加速度计 / 陀螺仪 / IMU / 里程计 概念笔记

> 配套文档：`技术笔记-运动学与IMU姿态估计.md`
> 目标：把这四个经常被混用的术语彻底分清。

---

## 0. 一句话区分

| 名字 | 它是什么 | 给你什么 | 物理量 |
|------|----------|----------|--------|
| **加速度计**（Accelerometer） | 一个**传感器**（芯片里的一部分） | 三个轴的加速度 | a (m/s²) |
| **陀螺仪**（Gyroscope） | 一个**传感器**（芯片里的另一部分） | 三个轴的角速度 | ω (rad/s) |
| **IMU**（Inertial Measurement Unit） | **加速度计 + 陀螺仪（+ 可选磁力计）** 打包的模块 | 上面两/三组 raw 数据 | a + ω (+ m) |
| **里程计**（Odometry） | **不是传感器，是一种"位姿估计方法"** | 机器人在世界里的位置和速度 | (x, y, θ) 和 (Vx, Vy, ωz) |

> 关键认知：**加速度计和陀螺仪是硬件；IMU 是装着它们的小模块；里程计是软件算出来的位姿轨迹。**

---

## 1. 加速度计：感受"力"，间接得到加速度

**原理**：MEMS 芯片里有个小弹簧 + 质量块，受力位移被电容/压阻测出来。

**它真正测的是**：

```
a_测 = a_真 − g
```

注意有个 `-g`：静止平放在桌上的加速度计读出来不是 0，而是 **+9.8 m/s²（指向上）**——因为它"感受到"桌面对它的支持力。这就是为什么静止时它能用来**测重力方向 → 反推 roll/pitch**。

**优缺点**：

- ✅ 静态准（重力向量长期稳定）
- ❌ 动态时把"运动加速度"和"重力"混在一起，分不清
- ❌ 振动噪声大

**用途**：

- 算 roll、pitch 角（用重力做参考）
- 检测自由落体、撞击
- 不能直接测速度，更不能直接测位置（积分两次会爆炸性漂移）

---

## 2. 陀螺仪：感受"旋转"

**原理**：MEMS 里一个振动质量块，旋转时会产生 Coriolis 力，被电容测出来。

**它测的是**：三轴角速度 `ω = (ωx, ωy, ωz)` rad/s。

**优缺点**：

- ✅ 短时积分准（短时间内角度变化可信）
- ✅ 不受平移加速度干扰
- ❌ 长时间会**漂移**（零偏不为 0 → 积分误差线性累积）
- ❌ 受温度影响大

**用途**：

- 短时高精度测姿态变化
- 必须配合加速度计/磁力计做长期姿态校正

---

## 3. IMU：把它们打包

**IMU = Inertial Measurement Unit**，惯性测量单元。

它**就是一个芯片或模块**，里面通常装：

| 类型 | 含义 |
|------|------|
| **6 轴 IMU** | 3 轴加速度计 + 3 轴陀螺仪 |
| **9 轴 IMU** | 6 轴 + 3 轴磁力计（指南针，给 yaw 做绝对参考） |
| **10 轴 IMU** | 9 轴 + 气压计（高度） |

你这台车上的 **QMI8658A 就是一个 6 轴 IMU**：

```
IMU = 加速度计 + 陀螺仪
      ↓
   raw a + raw ω
      ↓
   姿态融合（互补 / Mahony / EKF）
      ↓
   roll, pitch, yaw 或四元数 q
```

> ⚠️ 注意：**IMU 本身不输出"姿态角"**，它只给你 raw 加速度和 raw 角速度。把它们融合成 roll/pitch/yaw 是**你自己的算法**做的。
> 有些"高级 IMU"模块（比如 BNO055）内部带了融合芯片，会直接输出四元数，但 raw 数据本质还是 a + ω。

**IMU 的根本特性**：

- 它**不需要外部参考**（不需要 GPS、不需要看路标、不需要轮子着地）。
- 这就是"惯性"两个字的含义——只用牛顿定律和载体内部信息推算。

---

## 4. 里程计：完全不是传感器，是"算位姿的方法"

**里程计**（Odometry，词根 odo = 路）是一种**算法/估计**，目标是回答一个问题：

> "机器人现在在哪里，以多快的速度在动？"

输出通常是 ROS 里的 `nav_msgs/Odometry`：

```
position    : (x, y, z)
orientation : 四元数 (qx, qy, qz, qw)
linear vel  : (Vx, Vy, Vz)
angular vel : (ωx, ωy, ωz)
```

里程计不是某个传感器，而是**用某些传感器算出来的结果**。常见来源：

| 类型 | 输入数据 | 优点 | 缺点 |
|------|----------|------|------|
| **轮式里程计**（你这套车用的） | 编码器脉冲 | 短期精度高 | 打滑就废 |
| **视觉里程计**（VO） | 相机 | 不打滑 | 弱纹理/暗光下失效 |
| **激光里程计**（LO） | LiDAR 扫描匹配 | 精度高 | 计算量大 |
| **IMU 里程计** | IMU 二次积分 | 不打滑 | 漂得很快，几秒就不能看 |
| **融合里程计** | 上面几种 + EKF | 最鲁棒 | 复杂 |

### 4.1 你这台车的"轮式里程计"具体怎么算？

```
编码器脉冲 ─→ 单轮线速度 W_A/B/C/D
                 ↓ （正解矩阵）
            底盘瞬时速度 (Vx, Vy, ωz)        ← 这就是 RT_IX/IY/IW
                 ↓ （时间积分）
            位姿 (x, y, θ)                   ← 这就是 ROS 里的 /odom
```

所以**正解的产物 `RT_IX/IY/IW` 是里程计的"瞬时速度部分"**，把它积分一下就得到位置——这一步通常在上位机做。

## 5. 三者关系（用你这台车举例）

```
                    硬件层                  软件层
┌──────────────────────────────┐
│  QMI8658A (IMU 模块)          │
│  ┌────────────┐ ┌────────────┐│
│  │ 加速度计    │ │ 陀螺仪      ││
│  └─────┬──────┘ └──────┬─────┘│   raw a, ω
└────────┼───────────────┼──────┘     ↓
         │               │      ┌──────────────────┐
         └───────────────┴─────▶│ 上位机：Mahony   │── sensor_msgs/Imu (姿态)
                                └──────────────────┘
                                        ↓
                                ┌──────────────────┐
┌────────────┐                  │ EKF 融合         │── /odom (融合后的里程计)
│ 4 轮编码器  │── 正解 ──Vx,Vy,ωz─▶│ robot_localization│
└────────────┘                  └──────────────────┘
                                        ↑
                                轮式里程计 nav_msgs/Odometry
```

可以看到：

1. **IMU 和编码器都是"原始数据来源"**，但物理量完全不同（IMU 给惯性量，编码器给轮转速）。
2. **轮式里程计**是用编码器算出来的位姿。
3. **IMU 不是里程计**，但它可以**和里程计融合**得到更鲁棒的位姿（这就是 `robot_localization` 干的事）。

## 6. 最常见的混淆点

### 混淆 1："加速度计就是 IMU"

❌ 错。加速度计是 IMU 的**一部分**。说"读 IMU 数据"通常是同时读 acc + gyro。

### 混淆 2："IMU 能直接告诉我位置"

❌ 错，**绝对不能**。原因：

- 加速度计读到的是 `a_真 - g`，先要把重力分离出来（需要姿态）。
- 然后 `位置 = ∬ a dt`，**两次积分**——零偏 1 mg ≈ 0.01 m/s²，10 秒后位置漂 0.5 m，1 分钟漂 18 m。

所以纯 IMU **几秒钟内能用，几十秒就废了**。这就是为什么必须配合编码器或视觉。

### 混淆 3："里程计 = 里程表"

部分对。汽车的里程表（odometer）测累计里程；机器人里程计（odometry）测**完整位姿轨迹**，不只是距离。

### 混淆 4："IMU 不打滑就比编码器好"

不全对。IMU **不受打滑影响**这点确实优于编码器，但 **IMU 漂移比编码器累积误差快得多**。所以现实中是**两者融合**：编码器主导位置短期精度，IMU 主导姿态短期精度，靠 EKF 互相纠正。

## 7. 在你这台车上各司其职

| 角色 | 物理来源 | 在 STM32 上做了什么 | 上位机用它做什么 |
|------|----------|----------------------|------------------|
| 加速度计 | QMI8658A 内部 | 读 raw、坐标系映射 | 提供重力方向，纠正 IMU 姿态漂移 |
| 陀螺仪 | QMI8658A 内部 | 读 raw、零偏抵消、坐标系映射 | 短期高精度角速度 |
| IMU | 上述两个的组合 | 整体打包 → CAN 0x0121/0x0122 | Mahony/互补 → `sensor_msgs/Imu` |
| 里程计 | 4 路编码器 → 正解 | 算 RT_IX/IY/IW → CAN 0x0123 | 积分得 `/odom`，再 EKF 融合 IMU |

---

## 8. 一句话总结

> - **加速度计**：测"被推"的力。
> - **陀螺仪**：测"在转"。
> - **IMU**：把上面两个塞一个芯片里。
> - **里程计**：不是传感器，是用编码器（或视觉、激光、IMU）**算出来的"机器人轨迹"**。
>
> IMU 告诉你**车朝哪儿、转得多快**；
> 里程计告诉你**车走到哪、跑得多快**。
> 两个一起融合，才能得到鲁棒的"机器人在世界里的位置和姿态"。
